Getting Started

First Launch

When you first open Byblos, an onboarding wizard guides you through:

  1. Microphone permission — Byblos needs mic access to hear you.
  2. Download a model — Pick a Whisper speech model. Whisper Base (142 MB) is recommended to start.
  3. Ready — Byblos appears as a waveform icon in your menu bar. That's it.

Permissions

  • Microphone (required) — for voice recording.
  • Accessibility (recommended) — for typing text into other apps and the hold-to-record hotkey. Grant in System Settings → Privacy & Security → Accessibility.

Without Accessibility permission, Byblos still works — transcriptions are copied to your clipboard instead of typed directly, and you use the menu bar click to record instead of a hotkey.

Recording & Transcription

Starting a Recording

There are three ways to record:

  1. Left-click the menu bar icon — the simplest way. Click once to start, click again to stop (or just wait for auto-stop).
  2. Hold-to-record hotkey — hold Option (configurable), speak, release. Requires Accessibility permission.
  3. Transcript workspace button — open the Transcripts window and click the red record button at the bottom. Useful when you want to see the full transcription as it happens.

While Recording

  • The menu bar icon turns red.
  • A floating overlay appears at the bottom of the screen. This shows a preview of the transcription as you speak, along with a word count and elapsed time. Think of it as a heads-up display — a quick confirmation that Byblos is listening and understanding you.
  • For a full view of the transcription, open the Transcript workspace (right-click menu bar → Show Transcripts). The workspace shows the complete streaming text and saves everything.

Stopping

  • Left-click the menu bar icon again.
  • Release the hotkey (if using hold-to-record).
  • Click the overlay — the floating overlay is clickable.
  • Auto-stop — Byblos detects when you stop speaking and automatically stops recording after a few seconds of silence (enabled by default, configurable).

Where Does the Text Go?

After recording stops, the transcribed text goes to two places:

  1. Your active app — by default, text is typed directly into whatever app you were using before you started recording. Clicking the menu bar icon does not steal focus from your current app, so if you were typing in TextEdit, the transcription appears there.
  2. Transcript history — every transcription is automatically saved. Open the Transcript workspace anytime to review, search, edit, or export past transcriptions.

Tip: If text isn't appearing in your app, check that Accessibility permission is granted. Without it, Byblos falls back to copying text to your clipboard — just press ⌘V to paste. You can also set clipboard mode as the default in Settings → General → Output mode.

Undo

Made a mistake? Say "scratch that", "undo that", or "delete that" to delete the last transcription. Also available in the right-click menu → Undo Last.

Dictation Modes

Modes control how your speech is processed after transcription. Switch modes by right-clicking the menu bar icon → Mode.

Clean (default)

Removes filler words (um, uh, like, you know), fixes punctuation and capitalization. The best all-purpose mode — just speak naturally.

Email

Professional tone with paragraph structure. Speak your email naturally and Byblos formats it with appropriate paragraph breaks. Great for Mail, Gmail, Outlook.

Notes

Converts your stream of consciousness into bullet points. Speak freely about what's on your mind and get organized notes. Pairs well with Notes.app, Obsidian, Notion.

Translate

Speak in any language and get English text. Uses Whisper's built-in translation (99+ languages supported). Useful for multilingual meetings or translating thoughts.

Raw

Exact transcription with no processing. What Whisper hears is what you get — no filler removal, no formatting. Useful when you need a verbatim record.

Code Comment

Concise, technical output prefixed with //. Removes hedging language ("I think", "maybe"). Niche — designed for developers dictating comments while coding.

Agent EXPERIMENTAL

AI assistant mode — speak commands instead of dictation. Requires a local LLM. See Agent Mode.

Automatic Mode Switching

When enabled (Settings → General → "Auto-select mode based on app"), Byblos picks the right mode based on which app you're using:

  • Mail, Gmail → Email
  • Slack, Discord, Messages → Clean
  • VS Code, Xcode, Cursor → Code Comment
  • Notes, Obsidian → Notes
  • Everything else → your selected default

You can always override by manually selecting a mode.

Transcript Workspace

The Transcript workspace is your home for everything you've dictated. Open it from the right-click menu → Show Transcripts.

Layout

  • Sidebar — all transcriptions grouped by date (Today, Yesterday, This Week, Earlier). Click to select.
  • Detail panel — the full text of the selected transcription, along with metadata (duration, mode used, which app was active).

Recording from the Workspace

The red record button at the bottom lets you record directly into the workspace. While recording, the streaming transcription appears in the detail panel — useful when you want to watch the full text as you speak, rather than the compact overlay.

Search & Filter

Use the search field to find transcriptions by content. Use the mode dropdown to filter by dictation mode.

Editing

Click the text in the detail panel to edit it. Changes are saved automatically.

Exporting

Toolbar → Export → choose format:

  • Markdown — with headers, metadata, and dividers
  • Plain Text — just the text
  • JSON — structured data with all fields

Exports the selected transcription, or all transcriptions if none is selected.

File Transcription

Byblos can transcribe audio and video files — not just live speech.

  • Drag and drop a file onto the Transcript workspace
  • Or click Import Audio in the toolbar

Supported: WAV, MP3, M4A, FLAC, OGG, MP4, MOV, MKV. Non-WAV files are automatically converted using macOS's built-in audio tools.

The transcription appears as a new entry in your transcript history.

Custom Vocabulary

Settings → Vocabulary. Add word replacements for names, jargon, and abbreviations that Whisper consistently gets wrong.

Examples:

  • "jay bee" → "JB"
  • "byblos" → "Byblos"
  • "react" → "React"

Replacements are case-insensitive and applied after transcription. You can import/export your vocabulary as JSON to share or back up.

Agent Mode EXPERIMENTAL

Agent mode turns Byblos into a voice-controlled AI assistant. Instead of typing dictation, it interprets your command and takes action.

Requires a local LLM to be installed. Without one, Agent mode won't respond.

What It Can Do

  • "What's on my screen?" — reads the content of your current window
  • "Find files called readme" — searches via Spotlight
  • "Open Safari" — launches apps
  • "What's on my clipboard?" — reads clipboard
  • "What time is it?" — answers from context

Limitations

Agent mode is experimental. The local LLM (currently Qwen 7B) doesn't always parse commands correctly, and complex multi-step actions may not work reliably. It improves with better/larger models.

Local LLM Setup (Optional)

A local LLM powers smarter dictation cleanup and Agent mode. It runs in a separate helper process alongside Whisper — both use Metal GPU acceleration.

Installing

  1. Download a GGUF model file. Recommended:
    • Qwen2.5-7B-Instruct-Q4_K_M (~4.5 GB) — best balance of quality and speed
    • Phi-3.5-mini-instruct-Q4_K_M (~2.2 GB) — lighter alternative
  2. Place the .gguf file in:
    ~/Library/Application Support/Byblos/llm-models/
  3. Restart Byblos — it auto-detects and loads the model.

Requirements: Apple Silicon Mac with 16 GB+ unified memory for a 7B model. The LLM uses ~4-5 GB of GPU memory alongside Whisper.

Troubleshooting

No transcription / empty result

  • Speak close to the microphone
  • Check Settings → Audio → correct input device selected
  • Check Settings → Models → a model is downloaded
  • Try a larger model for better accuracy

Text not appearing in my app

  • Grant Accessibility permission: System Settings → Privacy & Security → Accessibility
  • Or switch to clipboard mode: Settings → General → Output mode → Copy to clipboard

Hotkey not working

  • Requires Accessibility permission (see above)
  • After updating Byblos, you may need to re-grant the permission

Agent mode not responding

  • Install a local LLM — see Local LLM Setup
  • Check ~/Library/Logs/Byblos.log for "[LLM]" entries

Logs

Byblos logs to ~/Library/Logs/Byblos.log. Include this when reporting issues.
